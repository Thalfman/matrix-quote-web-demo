### Issues Resolved

- Compare workspace ME-hours input crashed on `2,000` (Ben's reported repro, twice) — fixed in Phase 1, regression-tested.
- Hours by Sales Bucket displayed identical Total and Avg — fixed in Phase 1, fixture-based test added.
- Quote results gave no echo of user inputs ("did the model get my inputs right?") — fixed in Phase 1 with QuoteResultPanel recap.
- No interrogation path for chart bars or category labels — Phase 2 added drill-down tooltips + glossary.
- Insights download bundled `.json` ("How do I open a JSON file?" — Ben) — Phase 3 made the default bundle XLSX + README; engineer JSON download is opt-in.
- vercel.json cached the wrong path (`/demo-assets/models/` did not exist) — Phase 4 fixed the rules and removed the dead one.
- LFS-pointer guard silently skipped joblibs — Phase 4 made it hard-fail with the locked CONTEXT D-02 error.
- Jargon guard only covered DemoHome + DataProvenanceNote — Phase 4 extended to QuoteResultPanel + BusinessInsights + BusinessInsightsView, immediately catching a real "training data" leak.

### Issues Deferred

- **Manual UAT debt for Phases 2 + 3** — automated checks all green; manual smoke items deferred because user was away from PC at verify time. Captured in `02-UAT.md` and `03-UAT.md`. Non-blocking. Runnable later via `/gsd-verify-work 2` and `/gsd-verify-work 3`.
- **Code review Info findings (IN-01..IN-04)** in Phase 4 — byte-equivalence drift hazard, regex-flag inconsistency, under-asserted body-length invariant, glossary.ts forward-reference comment. Deferred per `--fix` scope (Critical+Warning only). Runnable via `/gsd-code-review 4 --fix --all`.
- **Quote persistence (PERSIST-01)** — UX-01 ships a same-day patch (input recap) that closes the most jarring information loss. Full persistence + edit + versioning lands in v2.

- **7 inline jargon-guards** outside the 3 surfaces locked into D-03. Phase 4 deliberately did not consolidate them (opportunistic dedup deferred). Worth a Wave-1-style sweep when a future phase already touches those surfaces.
- **The `xlsx` (SheetJS) bundle** is large enough to trigger a Vite chunk-size warning. Lazily bundled today; if Phase 3's insights pack becomes a high-traffic surface, code-splitting is the right next move.
