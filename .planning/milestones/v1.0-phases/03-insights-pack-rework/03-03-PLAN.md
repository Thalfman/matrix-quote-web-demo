## Risks / pitfalls

- **R1 — Existing `buildInsightsPackZip` test breaks by design.** The current test (`exportPack.test.ts:183-203`) asserts `projects.csv`, `portfolio.json`, `summary.md` are present. After T1, those assertions are wrong. The fix lives in T2 (new assertions for the new shape). **Important:** do not delete the test in T1 and add a new one in T2 — that's a hidden regression. Update the same `describe("buildInsightsPackZip")` block in T2 so the diff is reviewable.
- **R2 — `csv.ts` becomes orphaned by the bundle, but is still used elsewhere.** The on-screen "Download CSV" button in `TopProjectsTable` (per `csv.ts`'s `downloadCsv` export) still uses it. Do NOT delete `csv.ts`. Just stop importing `toCsv` in `exportPack.ts`. ESLint will flag the now-unused import; remove it.
- **R3 — Zip entry order matters for snapshot determinism.** The order MUST be `summary.md`, `business-insights.xlsx`, `README.md`. JSZip preserves insertion order in `zip.files` iteration. T2 asserts order via `Object.keys(zip.files)`.
- **R4 — XLSX-in-zip storage size.** SheetJS produces a zipped XLSX (XLSX is itself a zip). JSZip compressing a zip yields no additional reduction but no corruption either. JSZip's default options work; do NOT pass `compression: "DEFLATE"` with custom levels — defaults are fine.
- **R5 — `buildPortfolioWorkbook(portfolio, datasetLabel, generatedAt)` accepts an unused second/third argument today (plan 03-01 stubbed `_datasetLabel` / `_generatedAt`).** Pass `datasetLabel` and `generatedAt` from the zip caller anyway — when a future iteration uses them in the workbook, no wiring change is needed. T2 doesn't assert their use; it just confirms the workbook is present.
- **R6 — `buildSummaryMarkdown` is locked (CONTEXT D-05).** This plan calls it but does NOT modify the function body. The existing snapshot/contains assertions in T2 must still pass after T1.
- **R7 — Filename inside the zip vs zip filename.** Inside the zip: `summary.md`, `business-insights.xlsx`, `README.md` — fixed names per D-01. The zip filename itself is still `business-insights-{slug}-{date}.zip` via `packFilename` — unchanged from today (CONTEXT D-discretion).
