## v1 — Customer-trust fixes (shipped)

Goal achieved: when Ben (or any non-technical reviewer) re-tries the demo at https://matrix-quote-web-demo.vercel.app, nothing crashes, every chart says what it claims, and the insights pack is self-explanatory.

### BUG (Critical bug fixes)

- [x] **BUG-01**: Compare workspace ME-hours input accepts comma-formatted numbers (`2,000`, `2000`, `2,000.5`, empty, non-numeric) without crashing. Repro reported twice by Ben on the deployed demo. Root cause: `Number()` / `parseFloat()` on `"2,000"` returns `NaN`, downstream code did not tolerate `NaN`. **Outcome (Phase 1):** sanitiser strips commas, validates, and shows inline validation; never throws to the React error boundary. Vitest regression test covers all four cases.
- [x] **BUG-02**: Hours by Sales Bucket shows distinct Total and Avg values (whenever bucket has > 1 project). **Outcome (Phase 1):** chart now renders distinct values; fixture-based test confirms `total ≠ avg` for the multi-project case.

### UX (User experience polish)

- [x] **UX-01**: Quote inputs visible in the result panel — recap of every field the user submitted, shown alongside the estimate. Quick win for U1 from feedback. Defers full quote persistence (`PERSIST-01` in v2) — same-day patch that closes the most jarring information loss. **Outcome (Phase 1):** `QuoteResultPanel` echoes every submitted field next to the estimate.
- [x] **UX-02**: Hover on Complexity vs Hours bars reveals which projects make up that level. Drill-down tooltip listing project names + hours. **Outcome (Phase 2):** Recharts tooltip lists projects sorted desc by hours.
- [x] **UX-03**: Category-name tooltips with definitions (System Category, Sales Bucket, Vision Type, etc.) shown on hover anywhere a category label appears. **Outcome (Phase 2):** definitions sourced from `frontend/src/lib/glossary.ts` (8 terms). Tooltips are keyboard-focusable.

### DATA (Data correctness and build hygiene)

- [x] **DATA-01**: `vercel.json` cache header points at the correct model paths. Today it caches `/demo-assets/models/` which does not exist; actual paths are `/demo-assets/models_real/` and `/demo-assets/models_synthetic/`. **Outcome (Phase 4):** cache rule covers both; dead rule removed; immutable cache (`max-age=31536000`) on both.
- [x] **DATA-02**: `scripts/build_demo_static.py` LFS-pointer guard fails the build loudly when joblibs ship as LFS pointers. Today the guard "silent-skips" — broken builds can deploy. **Outcome (Phase 4):** guard now `_die()`s with locked CONTEXT D-02 error format. 2 new pytest cases (`test_lfs_pointer_triggers_hard_fail`, `test_missing_src_dir_does_not_hard_fail` — preserves the deliberate docs-only-deploy path).
- [x] **DATA-03**: Jargon guard extends to cover `QuoteResultPanel.tsx` and the Insights pages. Today only `DemoHome` and `DataProvenanceNote` are checked. **Outcome (Phase 4):** new canonical `frontend/src/test/jargon.ts` (single source of truth) + `jargon-guard.test.tsx` scanning `QuoteResultPanel`, `BusinessInsights`, `BusinessInsightsView`. The new test caught a real banned-token leak ("training data" in `DataProvenanceNote.tsx`) — Rule-1 in-scope copy fix to "those past projects" landed in the same phase. WR-01 follow-up added a 4th case for the synthetic `<DataProvenanceNote variant>`.

### INSIGHTS (Insights pack rework)

- [x] **INSIGHTS-01**: The downloadable insights pack does NOT include a raw `.json` artifact in the customer-facing bundle. Ben (the target persona) didn't know how to open it. **Outcome (Phase 3):** default bundle is now XLSX + README. The JSON path moved to a separate engineer-side "Download raw JSON (for engineers)" button on the Business Insights page.
- [x] **INSIGHTS-02**: The insights pack columns are self-explanatory. Today reviewer reported "Not sure what I'm looking at." **Outcome (Phase 3):** XLSX has Summary / Drivers / Raw / README sheets; bundled README explains every column. Notepad-style narrative content (which Ben praised) preserved unchanged.

## v2 — Workflow fit (deferred to next milestone)

Goal of v2: reshape the tool to match Ben's actual quoting workflow (multi-week revisions, multi-vision projects, ROM-quote-only path).

- **PERSIST-01**: Quotes are saved, listable, editable, and version-tracked across sessions. U1 done right per Ben's email: "we often revise quotes several times over many weeks or months." **OPEN DECISION:** browser-only (localStorage / IndexedDB) vs introducing a backend for the first time. Decide during v2 discuss-phase.
- **DATA-04**: The schema and ML pipeline support multiple vision types per project, each with its own hours contribution. Affects master Parquet generator, joblib bundle build, frontend quote form (multi-row vision picker), result-panel drivers (per-vision contribution), and Ben's data ingest format going forward.
- **ROM-01**: ROM-quote mode that takes only material cost as input and outputs an hours estimate, using a simple linear/average path. Customer validated as a high-value lightweight path.
- **BENCH-01** (optional, low priority): Benchmark our model's hours estimates against the Manager-built spreadsheet estimators Ben described as "extremely crude." Used as a marketing comparator, not as a training input. Out-of-scope decision is durable; this is just measurement.

## v3 — Manager out of the loop (deferred)

Goal of v3: complete the north-star reframe with real-data ingest and the AI Scope-Review sibling tool.

- **DATA-05**: Real-data ingest cycle is formalized. When Ben provides more real project data, there is a documented path to retrain the Real bundle and ship it without manual scripting. May or may not require a backend; will be decided after PERSIST-01's backend question.
- **AI-01**: AI Scope-Review tool ships as a sibling workspace to Single Quote / Batch / Compare / Business Insights. It consults a Manager-authored question library (with attribution: who said it, why) and uses an LLM to flag gotchas in customer-requested scope or in our proposed solution.

## Out of Scope (durable)

- **Manager hour-estimator spreadsheets as training inputs.** Customer deliberately excluded these. They may appear as benchmark comparators (`BENCH-01`) but never as features or labels.
- **Live retraining at runtime in this repo.** Static SPA architecture forces all model updates through build + redeploy. A v3 decision to introduce a backend may revisit this; until then, retraining is offline.
- **Synthetic company names made to look real.** Customer noted with humor; no work needed.
