| **D-09** | **Confidence-band widening rule = deterministic `bandMultiplier = 1.75` applied to the half-widths of the hero range and the combined-totals row.** Implementation: in `romEstimator.ts`, after the model returns `{p10, p50, p90}` per target, compute `upperHalf = (p90 - p50)`, `lowerHalf = (p50 - p10)`, then output `{p10' = max(0, p50 - lowerHalf × 1.75), p50, p90' = p50 + upperHalf × 1.75}`. - **No separate ROM-only model retrain.** ROADMAP phase notes verbatim: *"we are not retraining a separate ROM model."* `core/` stays read-only. The wider band is a UI rendering rule on top of the existing trained joblibs.

- **No `core/` change, no `_PREDICT_SHIM` change.** ARCHITECTURE.md "Editing core/ for a frontend feature" anti-pattern. ROM math lives in `frontend/src/demo/romEstimator.ts` exclusively.
- **No `schemaVersion: 3` bump.** The optional `mode` flag is additive. Phase 5 / Phase 6 saved quotes round-trip with `mode === undefined` → defaulted to `"full"` on read.
- **No ROM in the Compare workspace's input form.** Compare's BROWSE side is unchanged. Compare-side Save still writes a `mode === "full"` quote (that workspace is multi-project comparison, not single-project quoting). If Ben asks for a "compare two ROM quotes" surface in the future, it gets its own phase.
- **No "ROM-vs-full" comparator widget.** D-10 — the `/quotes` list satisfies SC-3.
- **No manager-rate dropdown.** A single `ROM_BASELINE_RATE_HOURS_PER_DOLLAR` constant in `romEstimator.ts` (D-05). Manager-authored hour-estimator spreadsheets are durably out (REQUIREMENTS.md Out of Scope; Ben S4 verbatim).

## Accessibility

- **`RomBadge`:** `<span>` with `aria-label="Preliminary estimate"`; visible text "Preliminary"; color contrast (text-ink #0D1B2A on bg-amberSoft #FAEBB5) = 11.7:1 (well over WCAG AA 4.5:1).
- **Form fields:** all labels are real `<label>` elements (re-uses Phase 5 `<Field>` primitive); errors render in `<p className="text-xs text-danger">` linked via `aria-describedby`.
- **Submit button:** `disabled` on the button is the source of truth; `aria-disabled` falls out of `disabled`. Pre-submit hint ("Fill in the four fields above to enable.") is rendered as plain text below the button (not an `aria-describedby` — would over-announce).
- **Sanity-check banner:** `role="status"` so screen readers announce when the wide-range banner appears post-submit.
- **Color-only signal forbidden:** the `RomBadge` carries the textual label "Preliminary" in addition to the amber chip color. No state is conveyed by color alone (matches Phase 5 UI-SPEC §"Color-only signal forbidden").
- **Focus-visible rings:** teal 2px ring at 2px offset (inherited from `globals.css`).
- **Reduced motion:** existing `prefers-reduced-motion` rule covers all transitions. Phase 7 adds NO new animations.
