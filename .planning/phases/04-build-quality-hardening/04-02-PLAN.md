# Plan 04-02 — LFS-pointer hard-fail (PLAN VIEW)

> Two-view plan. **PLAN VIEW** for human review. **CODE VIEW** below for the executor. Touches one production file + one test file. Disjoint from 04-01 and 04-03.

## Why this plan

Today `scripts/build_demo_static.py:_copy_model_bundle` (lines 149-204) does this when a joblib turns out to be an LFS pointer (file < 1024 bytes):

```python
if joblib_file.stat().st_size < 1024:
    skipped_lfs += 1
    continue
…
if skipped_lfs:
    print(f"WARN: {skipped_lfs} joblib(s) … were LFS pointers and were skipped. …", file=sys.stderr)
```

The build then continues, copies the few non-pointer files, emits the metrics JSON, and returns success. **The deploy ships with broken ML.** We've already had this bite once on Vercel where `git lfs pull` partially succeeded — the demo went up, the ML tool was dead, and we only noticed because Ben tried it.

DATA-02 fixes this by turning the silent-skip into a hard fail. The script already has a `_die(msg)` helper at lines 48-50 that prints `ERROR: {msg}` and `sys.exit(1)`. We use it. The message format is locked in CONTEXT D-02.

The pytest case (in the existing `tests/scripts/test_build_demo_static.py`) creates a fake `models_real/fake.joblib` of 100 bytes under `tmp_path`, monkeypatches `build_mod.DEMO_ROOT` to point at `tmp_path`, calls `_copy_model_bundle("models_real")`, and asserts `SystemExit` plus a stderr match.

## Risks / pitfalls

- **R1 — Vercel-build behavior change.** This is the intended change: a partial-LFS-fetch deploy now fails the build instead of shipping broken. That is the goal. If Vercel's `git lfs pull` is unreliable in some way we don't know about yet, we'll catch it on the FIRST deploy after this lands and have to tune the LFS pre-fetch in `scripts/vercel_build.sh:9-14`. That is the desired failure mode (loud, fixable) — not a regression.
- **R2 — Local builds without LFS-pulled assets.** A dev who cloned without LFS will now get a hard fail running `python3 scripts/build_demo_static.py`. The error message tells them exactly what to do (`run `git lfs pull` and re-build`). Acceptable.
- **R3 — The "src missing" branch is NOT changed.** Lines 161-167 (`if not src.exists()`) keep their current WARN-and-return-0 behavior. That branch represents a deliberate "build without ML assets" path (e.g., docs-only deploy where the operator deletes `demo_assets/models_*`). Confusing the two failure modes would break that path. CONTEXT D-02 locks this distinction.
- **R4 — pytest reliance on monkeypatch.** The new test case mutates `build_mod.DEMO_ROOT` at module attribute level. This works because `_copy_model_bundle` reads `DEMO_ROOT` at call time (line 160: `src = DEMO_ROOT / src_name`), not at import time. Verify by reading the current source before writing the test.
- **R5 — `_copy_model_bundle` also writes `OUT / f"model_metrics_{suffix}.json"` at the END (lines 199-202). With the hard-fail in front, the write never executes. That's correct: the build aborts before any output is produced, which is the safest state.

## Out of scope

```python
    for joblib_file in sorted(src.glob("*.joblib")):
```

```python
    for joblib_file in sorted(src.glob("*.joblib")):
        if joblib_file.stat().st_size < 1024:
            _die(
                f"LFS pointer detected at {joblib_file}; "
                f"run `git lfs pull` and re-build "
                f"(joblib bundle = {src_name})"
            )
        shutil.copy2(joblib_file, dst / joblib_file.name)
        count += 1
```

Also UPDATE the docstring at the top of `_copy_model_bundle` (lines 150-157 today). Replace the second paragraph (`If the source joblibs are LFS pointers ...`) with:

```
    If any joblib under src is an LFS pointer (size < 1024 bytes), this
    function calls _die() and the build aborts. Run `git lfs pull` and
    re-build to recover. This is intentional: a half-fetched LFS state

    The "src does not exist" branch above is a separate concern: that path
```

```python
def test_lfs_pointer_triggers_hard_fail(build_mod, tmp_path, monkeypatch, capsys):
    """A *.joblib file < 1024 bytes under DEMO_ROOT must abort the build via _die()."""
    # Set up a fake demo tree with one pointer-sized joblib in models_real/.
    fake_demo_root = tmp_path / "demo_assets"
    (fake_demo_root / "models_real").mkdir(parents=True)
    pointer_joblib = fake_demo_root / "models_real" / "me10_actual_hours.joblib"
    pointer_joblib.write_bytes(b"version https://git-lfs.github.com/spec/v1\n")  # 44 bytes
    assert pointer_joblib.stat().st_size < 1024, "test fixture must be a fake LFS pointer"

    # Redirect DEMO_ROOT and OUT so the function operates on the tmp tree.
    fake_out = tmp_path / "out"
    fake_out.mkdir()
    monkeypatch.setattr(build_mod, "DEMO_ROOT", fake_demo_root)
    monkeypatch.setattr(build_mod, "OUT", fake_out)

    with pytest.raises(SystemExit) as exc_info:
        build_mod._copy_model_bundle("models_real")

    # _die calls sys.exit(1)
    assert exc_info.value.code == 1, f"expected exit code 1, got {exc_info.value.code}"

    captured = capsys.readouterr()
    assert "LFS pointer detected at" in captured.err, (
        f"stderr did not contain expected LFS error message; got: {captured.err!r}"
    )
    assert str(pointer_joblib) in captured.err, (
        "stderr did not include the offending file path"
    )


def test_missing_src_dir_does_not_hard_fail(build_mod, tmp_path, monkeypatch, capsys):
    """When the entire src dir is missing, the function still returns 0 (CONTEXT D-02)."""
    # No models_real/ directory created — simulating a docs-only deploy.
    fake_demo_root = tmp_path / "demo_assets"
    fake_demo_root.mkdir()
    fake_out = tmp_path / "out"
    fake_out.mkdir()
    monkeypatch.setattr(build_mod, "DEMO_ROOT", fake_demo_root)
    monkeypatch.setattr(build_mod, "OUT", fake_out)

    # Must NOT raise SystemExit.
    result = build_mod._copy_model_bundle("models_real")
    assert result == 0, f"expected 0 from missing-src path, got {result}"

    captured = capsys.readouterr()
    assert "ML tool will be non-functional" in captured.err, (
        "missing-src branch should still emit its informational WARN"
    )
```

The second case is a regression guard — it locks CONTEXT D-02's "missing src is NOT the same as pointer detected" distinction.
