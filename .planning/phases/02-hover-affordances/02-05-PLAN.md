- **R1 — Recharts + ResponsiveContainer in jsdom.** Existing tests don't render the bar chart's hover behavior because jsdom doesn't size `ResponsiveContainer`. The drill-down logic (`bucketByComplexity`) is fully tested at the data layer in plan 02-03. The browser-level hover behavior is covered by the manual smoke gate in T2.
- **R2 — `<TooltipProvider>` mounting cost.** Each chart card and Field instance now mounts its own provider. Radix supports this without conflict, but if `npm run build` warns about chunk size (>500kB main chunk), check whether tree-shaking is pulling in the entire `@radix-ui/react-tooltip` for each provider. Likely fine; flag if not.
- **R3 — Pre-existing Vitest flakes.** Phase 1 verification passed 516/516 (per STATE.md). If anything unrelated to Phase 2 is flaky now, do NOT fix it here — open a separate todo and continue.
