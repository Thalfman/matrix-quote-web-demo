| Token | Value | Usage |

| 2 | 8px | Compact stack (status chip vertical) |

| 4 | 16px | Default element spacing, list-row vertical padding |
| 5 | 20px | Card inner padding (small) |

### Status-pill palette (PERSIST-05 ‚Äî Ben's verbatim five)

Five colored pills, one per workflow state. All use existing palette tokens; no new colors. Pills are 24px tall, `text-xs eyebrow` (uppercase Barlow), `px-2 py-0.5 rounded-sm`. Click target wraps a `min-h-[28px]` button.

| State | Background | Text | Tailwind Classes | Visual rationale |
|-------|------------|------|------------------|------------------|
| `draft` | `bg-line` (#E5E1D8) | `text-muted` (#5A6573) | `bg-line text-muted` | Neutral grey ‚Äî not yet active, no opinion |
| `sent` | `bg-tealSoft` (#D7ECF1) | `text-tealDark` (#177082) | `bg-tealSoft text-tealDark` | Teal = "in progress with the customer" ‚Äî matches v1.0 confidence-high tone |
| `won` | `bg-success/15` | `text-success` (#2F8F6F) | `bg-success/15 text-success` | Green = closed-won, the only outcome that should ever feel celebratory |
| `lost` | `bg-danger/10` | `text-danger` (#B5412B) | `bg-danger/10 text-danger` | Soft red ‚Äî terminal but not alarming. Doubles as the "soft-archive" cue (D-17 rationale) |
| `revised` | `bg-amberSoft` (#FAEBB5) | `text-ink` (#0D1B2A) | `bg-amberSoft text-ink` | Amber = "back in motion, this one moved" ‚Äî distinct from `sent` (teal = forward) |

**Important ‚Äî the five status labels are Ben's verbatim, lowercase, in chip text capitalized via CSS only:** chip displays as `Draft / Sent / Won / Lost / Revised` (the `eyebrow` class uppercases via `text-transform`, but the underlying string is "draft", "sent", "won", "lost", "revised" ‚Äî keeps Ben's verbatim contract enforceable in code).

**Hover/focus on the chip (clickable, advances state):** `hover:opacity-90` + `focus-visible:ring-2 focus-visible:ring-teal focus-visible:ring-offset-2`. Cycle order on click: `draft ‚Üí sent ‚Üí won ‚Üí lost ‚Üí revised ‚Üí draft`. Right-arrow / Enter / Space all advance one step. Holding Shift+click reverses one step (mouse-only convenience; not required for keyboard parity).

### "Save quote" dialog (D-12, D-14)

**Decision: modal dialog (centered overlay), NOT inline panel or slide-over.**

Rationale: (1) the "save quote" action is decisive and one-shot ‚Äî modal pattern matches the user's mental model of "I'm committing this estimate". (2) Slide-over from the right collides visually with the existing `ProjectDetailDrawer` (which is information-display, not data-commit). (3) Inline panel inside `QuoteResultPanel` would push the result down and create layout shift right when the user has just read the estimate. The modal preserves the result reading position and returns the user to it on Save / Cancel.

### Auto-suggested name format

**Decision: deterministic, plain, three-segment with `¬∑` separator.**

Format: `{salesBucket} {hourLabel} ¬∑ {visionLabel} ¬∑ {ISO date}`

Examples:
- `ME 800h ¬∑ Vision ¬∑ 2026-05-05`
- `EE 240h ¬∑ No vision ¬∑ 2026-05-05`
- `ME 1,200h ¬∑ 2D ¬∑ 2026-05-05`

Rules:
- `salesBucket` = first 2-3 chars of the discipline (ME / EE / ME+EE) per the existing input shape's `sales_bucket` field; falls back to `Quote` if empty.
- `hourLabel` = `{rounded estimate} h` with thousand-separator (`Math.round(result.estimateHours).toLocaleString()` + `'h'`).
- `visionLabel` = the form's `vision_type` value verbatim if non-empty; `No vision` if empty/null.
- `ISO date` = `YYYY-MM-DD` from `new Date()`; deterministic for the local timezone.
- All three segments joined with ` ¬∑ ` (middle dot, U+00B7, with surrounding spaces ‚Äî same character used in version history rows for visual consistency).
- Total length capped at 80 chars (matches the field validation max).

ESC closes (treated as Cancel). Backdrop click closes. The destructive button is on the RIGHT ‚Äî it is the "follow through" position; reverses standard "primary on right" only when the destructive intent is explicit, matching common modal dialog patterns.

## Interaction Behaviour

| Surface | Keyboard | Mouse | Touch |
|---------|----------|-------|-------|
| Quote row | Tab to focus, Enter / Space to open | Click anywhere (except chip / delete) opens detail | Tap |
| Status chip (clickable) | Enter / Space / Right-arrow advances; Shift+Right reverses | Click advances; Shift+click reverses | Tap advances (Shift unsupported on touch ‚Äî that's fine, sets the next forward state) |
| Sort buttons | Tab + Enter | Click | Tap |
| Modal (any) | ESC closes; focus trap inside; Tab/Shift-Tab cycles | Backdrop click closes | Backdrop tap closes |
| Save dialog name field | Enter submits; type-to-replace pre-selected text | Click to position cursor | Tap to focus |
| Restore button | Tab + Enter | Click | Tap |
| Delete icon (per-row) | Tab + Enter; row navigation does NOT happen on this focus | Click; `e.stopPropagation()` | Tap |
| Toast actions | Tab to focus, Enter activates | Click | Tap |

**Focus management on modal close:** focus returns to the trigger element. Save dialog ‚Üí focus returns to the "Save quote" button on `QuoteResultPanel`. Delete modal ‚Üí focus returns to the delete trigger (icon or detail-page button). The `useEffect` cleanup pattern from `ProjectDetailDrawer` is the reference implementation.

**Cross-tab broadcast UI cue:** when `BroadcastChannel('matrix-quotes')` notifies of a remote save / delete (D-15), the list re-renders silently. NO toast on remote events (would surprise the user in tab B). The version-history list inside an open detail page DOES re-render if the open quote's id matches.

## Accessibility

- **Status chip ARIA:** `aria-label="Status: {status}. Click to advance."` (button variant) or `aria-label="Status: {status}"` (read-only variant in version history).
- **Modal ARIA:** `role="dialog"`, `aria-modal="true"`, `aria-labelledby` referencing the title element. Focus trap mandatory.
- **Version-history list ARIA:** `<ul role="list">` (defensive ‚Äî Safari sometimes drops list semantics with `list-style: none`); items use `<li>`. Section labelled via `aria-labelledby="version-history-title"`.
- **List-row keyboard:** rows are `<button>` elements (not `<a>` ‚Äî they perform navigation but do not expose a URL on hover). Tab order is left-to-right within a row ‚Üí next row.
- **Color-only signal forbidden:** the status chip carries the textual label of the state in addition to color. No state is conveyed by color alone.
- **Focus-visible rings:** teal `#1F8FA6` ring at 2px offset 2px (already the global focus-visible from `globals.css`). Dark-mode overlay uses brighter cyan automatically.
- **Reduced motion:** existing `prefers-reduced-motion` rule in `globals.css` zeroes durations on all our animations (modal fade-in, hover transitions). Phase 5 adds NO new animations beyond the inherited `fade-in` / opacity transitions.
- **Screen-reader announcement on save:** sonner toasts are role="status" by default ‚Äî leave that. The "Mark as revised?" assist is a button inside the toast and gets focused on Tab.
